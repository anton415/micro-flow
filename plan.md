# План выполнения задания: Микросервисная архитектура в Kubernetes

## Обзор задания
Необходимо разработать демонстрационный проект микросервисной архитектуры с двумя сервисами (заявок и нотификаций), развернутыми в Kubernetes с использованием Ingress для внешнего доступа.

## Шаг 1: Подготовка инфраструктуры
- [x] Установить и настроить локальный Kubernetes кластер (Minikube или Kind)
- [x] Установить необходимые инструменты: kubectl, Docker, Helm (опционально)
- [x] Проверить работоспособность кластера

## Шаг 2: Проектирование архитектуры
- [x] Определить API для сервиса заявок (POST /requests)
- [x] Определить интерфейс взаимодействия между сервисами (HTTP или gRPC)
- [x] Спроектировать структуру данных для заявок и нотификаций

## Шаг 3: Разработка сервиса заявок (Request Service)
- [ ] Создать базовое приложение (Node.js/Python/Go)
- [ ] Реализовать REST API для приема заявок
- [ ] Добавить логирование
- [ ] Реализовать отправку запроса в сервис нотификаций
- [ ] Создать Dockerfile для контейнеризации

## Шаг 4: Разработка сервиса нотификаций (Notification Service)
- [ ] Создать базовое приложение
- [ ] Реализовать endpoint для приема нотификаций
- [ ] Добавить логирование полученных нотификаций
- [ ] Создать Dockerfile

## Шаг 5: Контейнеризация и оркестрация
- [ ] Собрать Docker образы для обоих сервисов
- [ ] Написать Kubernetes манифесты:
  - [ ] Deployments для сервисов
  - [ ] Services для внутренней коммуникации
  - [ ] Ingress для внешнего доступа
- [ ] Настроить Service Discovery между сервисами

## Шаг 6: Развертывание и тестирование
- [ ] Применить манифесты в кластере
- [ ] Проверить доступность сервисов через Ingress
- [ ] Протестировать полный цикл: отправка заявки → нотификация
- [ ] Проверить логи обоих сервисов

## Шаг 7: Документация и финализация
- [ ] Обновить README.md с инструкциями по развертыванию
- [ ] Добавить примеры использования API
- [ ] Протестировать сценарии отказоустойчивости (опционально)

## Рекомендации по началу
1. Начните с установки Minikube и проверки его работы
2. Выберите язык программирования (рекомендуется Node.js для простоты)
3. Создайте базовую структуру проекта с папками для каждого сервиса
4. Реализуйте сервисы по отдельности, начиная с Request Service
